## 飞书审批接入指南

### 1. 创建飞书应用
1. 打开 [飞书开放平台](https://open.feishu.cn/)，进入开发者后台
2. 点击「创建企业自建应用」，填写应用名称和描述
3. 在「凭证与基础信息」页面获取 **App ID** 和 **App Secret**

### 2. 开通审批权限
在应用的「权限管理」页面，搜索并开通以下权限：
- `approval:approval` — 审批应用（读写，包含创建实例和订阅事件）
- `contact:user.id:readonly` — 通过邮箱/手机号获取用户 ID（用于解析审批发起人）

提交权限申请，等待管理员审批通过后生效。

### 3. 创建审批定义
1. 打开飞书管理后台（开启开发者模式）：
   [飞书管理后台](https://www.feishu.cn/approval/admin/approvalList?devMode=on)
2. 点击「创建审批」
3. **设置表单控件**（控件 ID 在开发者模式下可见）：
   - 添加「单行文本」控件 → 用于显示审批标题（如"Project Publish: xxx"）
   - 添加「多行文本」控件 → 用于显示发布详情（工作流列表等）
   - 记录两个控件的 **widget ID**（如 `widget-abc123`），后续需要填入配置
4. **设置审批流程**：
   - 多级审批由飞书审批流程控制，添加所需审批节点数量
   - 如需项目负责人 → 空间管理员两级审批，添加两个审批节点
   - 审批人可选「固定审批人」或「发起时指定」
5. 发布审批定义
6. 获取 **Approval Code**：点击编辑该审批定义，从浏览器地址栏复制 `definitionCode` 参数值

> **重要**：使用飞书审批时，多级审批流转由飞书控制。系统只创建审批实例并接收最终结果（通过/拒绝），
> 不再使用系统内置的 PROJECT_OWNER → WORKSPACE_OWNER 两级流转逻辑。

### 4. 配置事件订阅（接收审批结果回调）
1. 在应用的「开发配置 → 事件与回调 → 事件配置」中：
   - 选择「将事件发送至开发者服务器」（Webhook 模式）
   - 设置请求地址为：`https://你的域名/api/approvals/callback/LARK`
   - 飞书会发送 URL 验证请求（`url_verification`），系统会自动响应 challenge
2. 在「添加事件」中搜索并订阅：`审批实例状态变更`（`approval_instance`）
3. 在「加密策略」中记录 **Encrypt Key** 和 **Verification Token**（如果设置了加密）
4. **发布应用版本**使配置生效
5. 系统会自动调用飞书订阅 API 激活事件推送（每个 approval_code 只需一次）

> 飞书事件推送失败时会按 15s → 5min → 1h → 6h 间隔重试（最多 4 次），系统已内置幂等处理。

### 5. 审批发起人
系统会在发布审批时自动获取当前操作用户的邮箱，通过飞书通讯录 API 解析为 open_id 作为审批发起人。
- 请确保系统中用户的邮箱与飞书账号邮箱一致
- 如果用户未配置邮箱，审批发起将失败

### 6. 填写配置
将上述信息填入下方对应字段，保存即可。保存后发布工作流时将自动在飞书中创建审批实例。
